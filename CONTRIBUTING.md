## 🚀 Как помочь

Вы можете внести вклад следующими способами:

1. **Баг-репорты** — найдите и опишите ошибку в [Issues](https://github.com/QuadDarv1ne/quantum-horizon/issues)
2. **Идеи функций** — предложите новую визуализацию или функцию
3. **Документация** — улучшите README, добавьте примеры
4. **Код** — исправьте баги или добавьте новые функции
5. **Перевод** — добавьте поддержку нового языка
6. **Дизайн** — предложите улучшения UI/UX

## 🛠 Начало работы

### 1. Форк и клонирование

```bash
# Форкните репозиторий на GitHub
# Затем склонируйте
git clone https://github.com/ВАШ_НИК/quantum-horizon.git
cd quantum-horizon
```

### 2. Настройка окружения

```bash
# Установка зависимостей
npm install

# Копирование .env.example
cp .env.example .env.local

# Запуск разработки
npm run dev
```

### 3. Создание ветки

```bash
# Создайте ветку для вашей функции
git checkout -b feature/awesome-feature
# или для исправления бага
git checkout -b fix/bug-fix
```

## 📝 Правила кода

### Стиль кода

- **TypeScript** — строгая типизация, никаких `any`
- **Именование** — camelCase для функций/переменных, PascalCase для компонентов
- **Компоненты** — функциональные с хуками
- **Форматирование** — используйте Prettier (настроен в проекте)

### Структура компонентов

```tsx
'use client'

import { useState } from 'react'

// ==================== CONSTANTS ====================
const SOME_CONSTANT = 42

// ==================== TYPES ====================
interface Props {
  value: number
}

// ==================== COMPONENT ====================
export function MyComponent({ value }: Props) {
  const [state, setState] = useState(value)

  return <div>{state}</div>
}
```

### Коммиты

Следуйте [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: добавить визуализацию гравитационных волн
fix: исправить расчёт времени в СТО
docs: обновить README
style: форматирование кода
refactor: оптимизация кэширования
test: добавить тесты
chore: обновить зависимости
```

## 🔀 Pull Request

### Перед отправкой

```bash
# Проверка линтером
npm run lint

# Сборка проекта
npm run build

# Тесты (если есть)
npm test
```

### Чеклист PR

- [ ] Код следует стилю проекта
- [ ] Нет console.log и отладочного кода
- [ ] Сборка проходит без ошибок
- [ ] Описаны изменения в PR
- [ ] Добавлены скриншоты (для UI изменений)

### Описание PR

```markdown
## Описание
Краткое описание изменений

## Тип изменений
- [ ] ✨ Новая функция
- [ ] 🐛 Исправление бага
- [ ] 📝 Документация
- [ ] ♻️ Рефакторинг
- [ ] 🎨 Стили
- [ ] ⚡ Производительность

## Скриншоты (если применимо)

## Чеклист
- [ ] npm run build проходит успешно
- [ ] npm run lint не показывает ошибок
```
